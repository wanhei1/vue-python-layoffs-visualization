## 🔄 数据转换核心逻辑

### 1. **数据筛选策略**

```
原始 1,418 条 → 筛选后 905 条
删除原因: 关键字段缺失 (Laid_Off, Date_layoffs, Industry)
筛选率: 63.8% 的数据被保留
```

### 2. **字段映射转换**

| 原字段 (Excel)                | 新字段 (JSON) | 转换逻辑                |
| ----------------------------- | ------------- | ----------------------- |
| `Location_HQ`                 | `Location`    | 🌍 地理位置标准化       |
| `Laid_Off`                    | `Num`         | 📊 裁员人数直接映射     |
| `Date_layoffs`                | `Time`        | ⏰ 日期格式标准化为UTC  |
| `Company_Size_before_Layoffs` | `Before`      | 👥 公司规模-裁员前      |
| `Company_Size_after_layoffs`  | `After`       | 👥 公司规模-裁员后      |
| `Stage`                       | `State`       | 💰 融资阶段标准化       |
| `Money_Raised_in_$_mil`       | `Money`       | 💵 融资金额清理特殊字符 |

### 3. **新增计算字段**

- `Scale`: 公司规模分类 (如 "1000-1500", "3500-5000")
- `level`: 规模等级数值 (1-15级，便于排序和筛选)
